# Bitter ‚Äî Like Coffee - Web-First Educational Social App

**Like Coffee** I love Coffee. Not actual bitterness.

![Bitter Screenshot](Screenshot%202025-11-03%20at%203.51.00 PM.png)

**Bitter** is a lightweight, front-end-only social application mockup created for **educational purposes**.  
It demonstrates layout, component structure, and basic event handling in **HTML, CSS, and JavaScript**, following accessibility and semantic web standards.

---

## üéØ Purpose

This project was designed as a **non-commercial classroom example** showing:
- Basic **HTML5 structure** (`header`, `aside`, `main`, `section`, `footer`).
- Responsive layout using **CSS Flexbox**.
- Simple UI interaction via **JavaScript event handling**.
- Client-side, no-backend simulation of user login, posting, and interface rendering.

---

## üß± Application Layout

The interface is divided into three main sections:

| Section | Description |
|----------|--------------|
| **Header** | Displays the application name (`Bitter`) and tagline. |
| **Sidebar (Left)** | Contains user info, list of other users, and a simple menu (Login, Logout, Settings). |
| **Main Feed (Center)** | Shows highlighted content, a post composer, and a scrollable content feed with ‚Äúspeech bubble‚Äù style posts. |
| **Scroll Bar Indicator (Right)** | A stylized vertical bar mimicking a scroll hint, matching the educational wireframe. |
| **Footer** | Lists APA-style references used for HTML, CSS, and JavaScript educational grounding. |

## üí° Features

- **No external dependencies** ‚Äì fully standalone HTML/CSS/JS.
- **Front-end login simulation** (no server required).
- **Local post handling** ‚Äì new posts appear in the in-memory feed.
- **Accessible labels** (`aria-label` attributes) for educational focus.
- **Responsive layout** ‚Äì works on desktop, tablet, and mobile screens.
- **APA-formatted references** for academic transparency.

1. Clone or download this repository.
2. Open `bitter.html` in any modern browser.
3. Interact with the app:

   - Explore highlighted content and user interactions.
